# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with this LaTeX Beamer presentation project for Bitcoin Austria.

## Project Overview

This repository contains LaTeX Beamer presentations for **bitcoin-austria.at**. Each presentation lives in its own subdirectory. The repo is self-contained: fonts (Blinker) and logos are bundled and need not be installed system-wide.

**Important**: When working with a specific presentation, always `cd` into its subdirectory first, or use `make -C <directory>` / the `PRESENTATION=keyword` pattern from the root.

**License**: Content is licensed under [MIT-Lizenz](https://opensource.org/licenses/MIT). Font (Blinker) is under SIL Open Font License 1.1.

## Build Environment

**Requires XeLaTeX** (for Blinker font via fontspec) and `latexmk`.

- `xelatex` must be in `$PATH`. On this machine it is symlinked at `~/bin/xelatex → ~/texlive/texlive/bin/x86_64-linux/xelatex`.
- The `/etc/LatexMk` on this machine overrides the `-xelatex` flag, so the makefile uses `-pdfxe` (not `-pdf -xelatex`).
- Each presentation directory must have a `.latexmkrc` containing `$pdf_mode = 5;` — this is set up automatically when you copy or scaffold a new presentation.
- Required TeX Live packages: `beamer`, `beamertheme-metropolis`, `fontspec`, `pgf`, `pgfopts`, `microtype`, `booktabs`, `xcolor`, `tikz`, `collection-latexrecommended`.

## Build Commands

All commands support fuzzy matching via `PRESENTATION=keyword`:

```bash
make [PRESENTATION=key]                          # Build PDF (quiet)
make build-verbose [PRESENTATION=key]            # Build with full LaTeX output (use when debugging)
make format [PRESENTATION=key]                   # Format source with latexindent
make clean [PRESENTATION=key]                    # Remove auxiliary files, keep PDF
make clean-all [PRESENTATION=key]                # Remove everything including PDF
make view [PRESENTATION=key]                     # Build and open PDF
make watch-quiet [PRESENTATION=key]              # Continuous rebuild on save
make screenshot-samples [PRESENTATION=key]       # Take sample screenshots
make screenshot-page PAGE=X NAME=desc [PRESENTATION=key]
make logos                                       # Regenerate logo PDFs from SVG (inkscape + pdfcrop)
make help                                        # List available presentations and commands
```

**Workflow**: Edit `.tex` → `make format PRESENTATION=key` → `make build PRESENTATION=key`

If the build fails silently, always run `make build-verbose` to see the full LaTeX log.

### Per-Presentation Directory (e.g., `2026-meine-praesentation/`)

```
2026-meine-praesentation.tex   # Main source — same name as directory
pix/                           # Presentation-specific images
generated-count.tex            # Auto-generated (gitignored)
```

**Naming convention**: The `.tex` file must have the same name as its directory.

**Required in every presentation directory**: a `.latexmkrc` file with:
```perl
$pdf_mode = 5;
```
This forces XeLaTeX regardless of system-level latexmk configuration.

## Style Package: `bitcoin-austria.sty`

Include it after the Metropolis theme:

```latex
\documentclass{beamer}
\usetheme[progressbar=foot]{metropolis}
\usepackage{bitcoin-austria}
```

### Macros

**Title setup:**
```latex
\bitcointitle{Haupttitel}{Untertitel}
\bitcointitlegraphic{pix/titelbild.jpg}   % optional hero image
```

**Rounded image:**
```latex
\roundedimage{pix/bild.jpg}{0.4\textwidth}{1}   % 1:1 aspect
\roundedimage{pix/bild.jpg}{0.6\textwidth}{2}   % 2:1 aspect (landscape)
```

**Footnote pattern** (overlay-safe):
```latex
\footnotemark~in text
\only<2->{\footnotetext[1]{Quelle: ...}}
```
